# Пошаговая инструкция по внедрению SEO Phase 2

## Быстрый старт (20 минут)

### Шаг 1: Schema.org для товаров (10 минут)

1. **Открыть файл:** `schema_product_inmunoflam_30.html`
2. **Скопировать** весь код между `<script>` и `</script>`
3. **Найти** шаблон товара: `www/templates/shop1/shop.commodity.full.tpl`
4. **Вставить** код перед закрывающим `</head>`
5. **Сохранить** и загрузить на сервер

**Для других товаров:**
- Изменить `name`, `image`, `description`, `price`, `url`
- Использовать переменные CMS: `{$product_name}`, `{$product_price}` и т.д.

---

### Шаг 2: Schema.org для статей (5 минут)

1. **Открыть файл:** `schema_article_template.html`
2. **Скопировать** весь код
3. **Найти** шаблон статьи: `www/templates/shop1/content.full_page.tpl`
4. **Вставить** код перед закрывающим `</head>`
5. **Сохранить** и загрузить на сервер

---

### Шаг 3: Оптимизация .htaccess (5 минут)

1. **Открыть файл:** `htaccess_optimized.txt`
2. **Скопировать** весь код
3. **Открыть** файл `.htaccess` на сервере
4. **Добавить** скопированный код **в конец файла**
5. **Сохранить**

**Важно:** Не удаляйте существующий код в .htaccess, только добавьте новый!

### Этап 3: Оптимизация изображений (1-2 недели)

**3.1. Конвертация в WebP**
- [ ] Открыть https://squoosh.app/
- [ ] Загрузить изображение
- [ ] Выбрать формат WebP
- [ ] Качество: 80-85%
- [ ] Скачать
- [ ] Заменить на сервере

**3.2. Добавление lazy loading**
- [ ] Найти все `<img>` теги в шаблонах
- [ ] Добавить атрибут `loading="lazy"`
- [ ] Добавить `width` и `height`
- [ ] Сохранить

**Пример:**
```html
<!-- Было -->
<img src="image.jpg" alt="Описание">

<!-- Стало -->
<img src="image.webp" alt="Описание" loading="lazy" width="800" height="600">
```

---

### Этап 4: Минификация (опционально, 1-2 часа)

## Частые вопросы

**Q: Когда Schema.org появится в поиске?**  
A: Обычно 1-4 недели после индексации страницы.

**Q: Что делать, если сайт сломался после .htaccess?**  
A: Удалить добавленный код и проверить синтаксис.

**Q: Как часто публиковать статьи?**  
A: Оптимально 1-2 статьи в неделю.

**Q: Обязательно ли конвертировать в WebP?**  
A: Нет, но это даст значительное ускорение.

**Q: Можно ли не минифицировать CSS/JS?**  
A: Да, это опционально. Основной эффект от кеширования и Gzip.
